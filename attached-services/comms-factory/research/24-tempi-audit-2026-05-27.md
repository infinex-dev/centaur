# 24-Tempi classification table — canonical audit

**Date:** 2026-05-27

**Mandate:** audit, do not rewrite code. Return the audited table; we decide what goes in the Director prompt vs. a longer human reference.

## TL;DR

1. **Names are correct.** All 24 tempo names in the current code match Mirodan's canonical Variation labels (Commanding, Practical, Sombre, Irradiant, etc.). No renames needed.
2. **The structural shape is right.** 4 Variations × 6 Inner Attitudes = 24. Mirodan uses "Variation" and "tempo" interchangeably. The 24-row table is a faithful synthesis.
3. **`Adream outer` is mislabelled.** Adream is a baseline Inner Attitude (Weight + Flow), not an outer-only Action Attitude. Mobile, Remote, Awake are the three outer-only attitudes. Strip "outer" from the four Adream rows.
4. **8 motor arrows point the wrong way structurally.** Strict prep→release ordering (`Sustained → Quick`) is canonical ONLY for the four classic Working-Action partners (Pressing→Punching, Wringing→Slashing, Gliding→Dabbing, Floating→Flicking). Near and Awake have no Flow axis — their two-motif rows co-exist on the same Time pole and should use `/` not `->`. Mobile already uses `/` correctly. **Eight Near + Awake rows currently lie about temporal order.** (See Critical Fixes §3.)
5. **About half the operator hints drift into theatre-ish slop** (`institutional landing`, `manifesto-force`, `wall-coming-down thesis`, `quiet vision-arrival`). Mirodan never used this vocabulary. The 24-Outer-Interpretation quotes from Vol 2 are the canonical shorthand — short, grounded, and usable.
6. **Recommended schema change:** split `classification_hint` into two fields — `canonical_shorthand` (Mirodan quote, locked) and `operator_hint` (modern-prose translation, mutable). Director receives both, knows which is canon.
7. **Single hint per row is correct.** No need to double rows for introvert/extravert (that split lives at the Subconscious Motif level on the Flow axis, not at the Variation level) or for prep/release (that's an internal arc within one Working-Action motor, not two scoring units).

## 1. Canonical anchors (terminology used below)

**Six Inner Attitudes** (Mirodan p. 374, p. 487/409/453/503/432/474):

| Attitude | Effort-factor pair | Inner / Outer? |
|---|---|---|
| **Stable** | Weight + Space | Inner (baseline-eligible) |
| **Near** | Weight + Time | Inner (baseline-eligible) |
| **Adream** | Weight + Flow | Inner (baseline-eligible) |
| **Mobile** | Time + Flow | Outer-only (Action Attitude under Stress) |
| **Remote** | Space + Flow | Outer-only (Action Attitude under Stress) |
| **Awake** | Space + Time | Outer-only (Action Attitude under Stress) |

The 24 Variations = 4 Variations × 6 Attitudes. Mirodan never bundles them as a single master table; she presents the four Variations of each Attitude inside Ch II of Vol 2.

**Mirodan's column structure per Variation:** `Variation (factor pair) | Label | Subconscious Motifs | Outer Interpretation`. The current code keeps `Variation`, `Label` (=name), and `Subconscious Motifs` (=motor). It collapses `Outer Interpretation` into a free-form `classification_hint`. That's the field this audit reworks.

**Working Actions (Laban basic Effort Actions, Mirodan p. 341):**

| Working Action | Weight | Space | Time |
|---|---|---|---|
| Floating | light | flexible | sustained |
| Flicking | light | flexible | quick |
| Gliding | light | direct | sustained |
| Dabbing | light | direct | quick |
| Wringing | strong | flexible | sustained |
| Slashing | strong | flexible | quick |
| Pressing | strong | direct | sustained |
| Punching | strong | direct | quick |

**Prep→Release pairings (Mirodan p. 347-348, Carpenter):** Sustained = inner preparation (introvert / past), Quick = outer execution (extravert / future). The four canonical pairs are:

- Pressing → Punching
- Wringing → Slashing
- Gliding → Dabbing
- Floating → Flicking

These are the ONLY pairings where `->` is structurally correct. Same-Time-pole pairs (two Sustained, or two Quick) co-exist within a Variation — Mirodan writes them with `/`, not `→`.

## 3. Critical fixes summary

Three classes of issue surfaced. Severity ordered.

### A. Motor arrow direction — 8 rows (structural, factually wrong)

Strict prep→release ordering (`->`) is canonical ONLY for the four classic Working-Action pairs. The current code applies `->` to rows where both motifs sit on the same Time pole, which is a category error: prep→release implies Sustained-then-Quick, but `Punching / Slashing` are both Quick.

| Row | Current | Should be |
|---|---|---|
| materialistic | `Punching -> Slashing` | `Punching / Slashing` |
| human | `Floating -> Gliding` | `Floating / Gliding` |
| warm | `Pressing -> Wringing` | `Pressing / Wringing` |
| cool | `Dabbing -> Flicking` | `Dabbing / Flicking` |
| acute | `Punching -> Dabbing` | `Punching / Dabbing` |
| doubting | `Floating -> Wringing` | `Floating / Wringing` |
| certain | `Pressing -> Gliding` | `Pressing / Gliding` |
| uncertain | `Flicking -> Slashing` | `Flicking / Slashing` |

Mobile (lines 103-106) already uses `/` correctly.

**Downstream impact:** `validator.ts:478-482` checks `motor[0]` (Sustained) vs `motor[1]` (Quick) for prep-hierarchy validation. That check is meaningful only for the four classic Sus→Q pairs. If the motor field changes to use `/` (or to a tuple/array signalling co-existence), the validator's prep-hierarchy enforcement needs to skip co-existence rows. Currently it's likely silently mis-firing on these 8 rows.

### B. Attitude label `Adream outer` — 4 rows (categorical mislabel)

Adream is an **Inner** Attitude (Weight + Flow), baseline-eligible like Stable and Near. Only Mobile, Remote, and Awake are outer-only Action Attitudes. Strip `outer` from sombre, overpowering, diffused, irradiant.

| Row | Current | Should be |
|---|---|---|
| sombre | `Adream outer` | `Adream` |
| overpowering | `Adream outer` | `Adream` |
| diffused | `Adream outer` | `Adream` |
| irradiant | `Adream outer` | `Adream` |

**Downstream impact:** attitude is display-only (no programmatic gating per Agent B grep). Low blast radius — but reading it as canon is misleading the Director's classification.

### C. Hint text — ~12 of 24 rows (drift from canon)

Hints that introduce non-canon vocabulary (operator metaphors, product-marketing prose, theatre-ish coinage):
- `commanding` — "institutional landing"
- `sombre` — "wall-coming-down thesis"
- `overpowering` — "manifesto-force", "risky if overused"
- `diffused` — "post-launch reflective wonder", "vision-arrival"
- `irradiant` — "future-warmth"
- `materialistic` — "body-forward contact"
- `warm` — "held relational force"
- `unacknowledged` — "hurried cover" (overshoots into aggression)
- `revealed` — "release into visibility"
- `concealed` — "withheld weight" (factor-shape repeated, not Mirodan's 'frigid self-assertion')
- `altruistic` — "warm-distant handoff", "for-you architecture"
- `sociable` — "partner credit"
- `unsociable` — "sidesteps discourse"

These don't break execution (the hint is informational text in the Director prompt, not parsed) but they pollute the Director's classification vocabulary. They lead to "decisive pressure, institutional landing" being treated as canonical when Mirodan's actual canon is "Commanding demonstration or acceptance of a 'bold resolve'".

**Recommended schema change:** add a `canonical_shorthand` field (locked, quoted from Mirodan Vol 2) and rename `classification_hint` to `operator_hint` (explicitly mutable, marked as operator shorthand). Both go into the Director prompt with clear labels.

## 4. Answers to the six audit questions

### Q1. Which current hints are grounded enough to keep?

Roughly half. The cleanest current hints — `practical`, `cool`, `doubting`, `uncertain`, `acute` — are within paraphrase distance of Mirodan's Outer Interpretation. Even these can be tightened, but they're not slop.

The other ~12 (listed in §3C) introduce vocabulary Mirodan never used. Replace.

### Q2. Which are vibes and should be deleted/replaced?

All 13 phrases listed in §3C. The ones with the largest semantic drift are `manifesto-force` (overpowering), `wall-coming-down thesis` (sombre), `for-you architecture` (altruistic), and `post-launch reflective wonder` (diffused) — these are not paraphrases of Mirodan, they're operator coinage from comms/product domain. None of them anchor classification; they're prompt clutter that can mislead.

### Q3. Are there Mirodan/Carpenter/Malmgren/Laban phrases we should use instead?

Yes — Mirodan's **Outer Interpretation** quotes (one per Variation, listed in every row's `canonical_shorthand` field above). These are 8-15 words each, locked, directly quotable. They're the canonical shorthand the table currently lacks. Adding them as a separate field gives the Director a grounded fallback while operator_hint remains mutable for modern-prose readability.

Carpenter (footnoted at p. 348 in Vol 2): *"[Pressing, Wringing, Floating, Gliding] have a sense of past; [Punching, Slashing, Dabbing, Flicking] have a sense of future."* This is useful framing for the motor field but doesn't belong per-row.

### Q4. Do any tempi need multiple hints (intro/extravert, prep/release, lining/outer action)?

**No.** Three axes were considered:

- **Introvert / extravert:** lives at the Subconscious Motif pair level on the Flow axis only — Bound = introvert, Free = extravert (Mirodan Vol 2 p. 469). It cross-cuts the 24 Variations but doesn't double them. Stable, Near, Awake have no Flow axis and therefore no intro/extravert distinction. Adream, Mobile, Remote already encode the Bound (introvert) and Free (extravert) versions as separate Variations within the 4-per-attitude set. **Already in the 24.**
- **Preparation / release:** an internal temporal arc within one Working-Action motor (Mirodan p. 347: "this order cannot be reversed"). Mirodan treats prep+release as **one** scoring unit with two phases, not two units. **Single row per Variation is canonically correct.**
- **Lining / outer action:** lining is a per-character secret Inner Action (Vol 2 pp. 554-555). It's a property of the *character* (e.g., Infinex), not of the Variation. Infinex's lining is already separate (`infinex.ts` carries `lining` per tempo). **Don't put lining in the taxonomy table; keep it in the placement spec.**

Single hint per row is correct.

### Q5. Is the 24-tempo table itself structurally correct?

**Yes, with the three fixes in §3.** The 4 × 6 = 24 partition is faithful to Mirodan (4 Variations × 6 Attitudes). Names are correct. Factor shapes are correct. Working-Action motors are correct identities; only the arrow direction needs fixing on 8 rows.

The only structural gap worth flagging is that the taxonomy doesn't carry Mirodan's **Outer Interpretation** quote per row — that's the missing field, not a missing row.

### Q6. Should the Director receive terse shorthand, longer explanatory prose, or both?

**Both, in distinct fields.** Recommended Director prompt shape per row:

```
- commanding: Stable; Strong + Direct; motor Pressing -> Punching
    canon: "Commanding demonstration or acceptance of a 'bold resolve'" (Mirodan Vol 2 p. 494)
    hint: Bold-resolve pressure landing as a single direct decision; no ornament around it.
```

Why both:

- **Canon line** is the locked anchor. The Director can quote it as evidence when it's unsure ("the prose matches Mirodan's 'bold resolve' framing").
- **Hint line** is the modern-prose translation. Useful for fast classification of contemporary crypto/comms prose where 1997 theatre language doesn't map directly. It's also where domain-tuned shorthand can evolve without polluting canon.

The current single-field approach forces the operator to either keep Mirodan's quote (alien to crypto prose readers) or replace it with modern shorthand (drifts off-canon). The two-field approach has both.
