# extension — sample release card

## Release card (metadata)
- name: Infinex Extension
- ship date: live on Chrome, Brave, Edge; Firefox coming soon
- audience: existing Infinex account holders who want to use their account on apps that aren't Infinex (Hyperliquid web UI, Aave, Synthetix, etc.); MetaMask refugees; anyone with a "20-wallet problem"
- product page: https://infinex.xyz/extension
- deployed facts:
  - "Connect with Infinex to your favorite onchain apps"
  - Chrome / Brave / Edge available; Firefox coming soon
  - Supports Hyperliquid, Aave, Synthetix; more protocols on the way
  - Swidge built in (cross-chain swap from inside the extension)
  - Passkey login on any device
  - Multi-wallet support (choose which wallet connects)
  - Explicit permission display per dapp
  - Connection management UI
  - Cross-protocol position view
  - Asset moves between your own wallets

## 3 candidate data points to hero
1. **3 browsers shipped, 1 coming** — the install-surface count. Visual: four browser icons, three filled, one outlined.
2. **N protocols supported / N positions visible** — the live counter of "things you can do with one passkey" — anchored on real protocol integrations (Hyperliquid + Aave + Synthetix at minimum).
3. **"Connect" event delta** — every dapp connection that used to require seed phrase / hardware / multiple approvals now requires one passkey tap. The data is the *removed* count: 0 seed phrases, 0 confirmation popups, 0 chain switches.

## Voice notes
Extension is the most adjacent-to-MetaMask product, which is the one place "exchange/wallet potshot" temptation will be strongest — DO NOT take it. The win is structural (passkey > seed phrase) and self-evident; let the comparison live in the user's head, not in our copy. The wry register can lean into the absurdity of seed-phrase UX *as a category*, never at a named competitor. Confident tone on browser coverage; humble tone on "more protocols on the way" — don't over-promise the roadmap.
