My read

Ship Variant C now. It's the strongest decontamination move and Claude has rich prior on Stanislavski/Mirodan terms — the canon reference does real work without needing an example. Variant B is the safe compromise if C feels too radical (it keeps a template-with-placeholders, which gives a grammar without a frame).

Don't pretend we can reason the right answer all the way. Run the A/B/C test next as the empirical answer.

Overnight jobA/B/C test plan

  1. Build a release corpus — 8–10 input scenarios spanning card kinds (fiat onramp, chain add, yield vault, partnership, maintenance, launch tier, split, wry data card).
  2. Three branches of card.ts — Variant A (token swap), Variant B (structural placeholder), Variant C (no example, canon ref).
  3. Ground each release through the fact-grounder LLM under each variant — capture the proposed through_action and lining per (input × variant).
  4. Measure per variant:
    • Lexical diversity: token-level Shannon entropy across proposed values
    • Anchor-token presence: does any variant's specific tokens (wall / evaporate / precondition / etc) appear above baseline rate
    • Frame clustering: embed proposed values, measure intra-variant cluster tightness — lower = more diverse frames
    • Validator pass rate: does the proposed through_action survive downstream validation
    • Operator readability check: are the proposed values intelligible / actionable to a human reviewer
  5. Pick winner on the diversity-vs-quality Pareto front.

Parallel agent: ~2–3 hours of wall time to build the harness + run all three variants + emit a results doc. Genuinely overnight-runnable.

Decision point

Pick one path now:

  • Ship Variant C immediately (highest theoretical decontamination; no measurement) — I apply now, you watch for regressions.
  • Ship Variant B immediately (safer, keeps a template, still defeats 3 of 4 anchors) — same flow as above.
  • Run the A/B/C test first — spin up the harness, measure empirically, then ship the winner. Slower but evidence-backed.
  • Ship C now AND queue the A/B/C test — get the strongest move in immediately; validate empirically afterward; revert to B (or back to A) if C measurably underperforms.
