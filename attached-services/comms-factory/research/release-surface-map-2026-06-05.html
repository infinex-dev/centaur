The simplification: one changelog blog file drives three surfaces — the blog page, the in-app what’s-new popout, and the mobile CTA. So the actual one-button PR is small: a changelog .md + the roadmap tick, optionally a feature card and an app-alert/carousel entry. The modal and the on-ramp enablement are flags (ops), not the PR; the X posts are manual via Typefully.
