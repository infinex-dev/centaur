# Codex spec — Image brief as a GENERATED SURFACE in the package

**Supersedes** the over-engineered approach in `docs/plans/2026-06-10-001-feat-image-briefs-segment-plan.md` and `docs/brainstorms/image-briefs-segment-requirements.md`. Operator rejected the "separate artifact type / separate tab / templated" design. This is the corrected, confirmed design.

## What was already removed (don't rebuild it)
- The separate "Image briefs" tab in `harness/components/design/PackageTabs.tsx` — gone.
- `harness/components/design/ImageBriefs.tsx` — deleted.
- The page wiring in `harness/app/cards/[id]/page.tsx` — reverted.
- `src/brief-generator.ts` still contains a DETERMINISTIC string-template `buildImageBriefs` — **this is the thing to replace with real generation.**
- `src/brief.ts` (kinds, `CROP_SPECS`, `OnImageText`, scene-template constants) **stays** — reuse it as data.

## Confirmed design (operator's words)
> "Who generates them? … why they're not just another part of the package … there's no button for me to click like copy."

The image brief is **ONE surface in the Package grid**, alongside `x` / `x-thread` / `web` / `blog` / `carousel`:
1. **Generated by an LLM** at Generate time (grounded on the card's `deployed_facts`), NOT templated.
2. Rendered as a surf-card in the existing grid with the **same copy/edit affordance** the other surfaces have (the `SurfaceEditor` copy button + editable text; edits persist as a final pick).
3. Covers the 3 standing kinds (`hero`, `in-app-mobile`, `feature-detail` from `src/brief.ts`) in ONE brief block — scene + on-image text + crop note per kind. ONE surface, not three.

## Acceptance
- Running Generate on a card yields an **image-brief** surface in the Package grid.
- Its body is LLM-generated and grounded — the scene references the actual launch (uses `deployedFactClaims(card)` / `outward_product_name` / `feature_states`), not a mail-merge of the title.
- It has the copy button + is editable like the other surfaces; edits persist.
- On-image lines pass `validator.validate()` (slop gate); a failing line is dropped, not shipped.
- It does NOT run the movement/beat/director/structured gates (it is art direction, not voiced copy).

## Implementation

1. **Channel** — add `"image-brief"` to the `Channel` union (`src/generator.ts:110`). Add a `CHANNEL_GENERATION_PROFILES` entry with art-direction guidance. Keep its output as **readable text** so `SurfacePreview`/`SurfaceEditor` render + copy it for free (optionally also a structured payload).

2. **Generator** — in `src/brief-generator.ts`, REPLACE the templated `buildImageBriefs` with `generateImageBrief(card, opts)`: a dedicated LLM call (client param + stub/live, mirroring `generateActorAttempt` in `src/actor-director.ts` — `createMessageWithRetry`, `assertLiveActorDirectorMode`). It is NOT routed through the movement-actor. Ground on `deployedFactClaims(card)` + `outward_product_name` + `feature_states`. For each `STANDING_BRIEF_KINDS` kind: a grounded scene, an on-image line (or none) run through `validate()` (drop on fail), and the crop note from `CROP_SPECS[kind]`. Return readable brief text (the copyable artifact) + optional structured JSON. Keep `src/brief.ts` as the source of kinds/crop-specs/on-image validation.

3. **Validation gating** — exempt `image-brief` from movement/beat/structured/director validation in the orchestrator/validator (`src/actor-orchestrator.ts` `validateCandidate`, `structureIssues`, `CHANNEL_MAX_LEN`, director audit). Only the slop validator applies, and only to its on-image lines.

4. **Harness surface meta** — add `image-brief` to `harness/lib/surfaces.ts` (`SurfaceKind` + `SURFACE_META`: label "Image brief", role "art-direction for the designer"). Ensure it flows into `expectedChannels` / `displayChannels` and `buildPackageView` rows so it renders as a surf-card in the grid. `SurfacePreview` may need a brief-aware branch (else plain text is acceptable).

5. **Generation flow + persistence** — when the operator generates a package (`harness/app/actions/*` → `emit-process` / actor run), also generate + persist the image-brief as a `candidates` row (channel `"image-brief"`) so it appears in the grid and is editable/shippable via the existing `final_picks` path (channel is free text). Run it alongside the copy generation.

6. **Tests** — unit-test `generateImageBrief` in stub mode: scene grounded in the card subject; a cliché on-image line is dropped; all 3 kinds covered. Update `src/__tests__/brief.test.ts` (currently tests the removed `buildImageBriefs`). Keep the full suite green; `tsc --noEmit` clean for both `src/` and `harness/`.

## Constraints
- ONE surface in the grid, not three.
- Don't auto-post; image-brief is not an outward posting channel.
- Mirror existing patterns (actor stub/live client, surfaces meta, package-view, SurfaceEditor).
- Don't regress the copy channels.
